# Architecture Decision Records

Short records of the non-obvious architectural choices that shape ZeroDark.
Each ADR follows the same template:

- **Status** — Accepted / Superseded / Deprecated
- **Context** — What problem made this decision necessary
- **Decision** — What we chose
- **Consequences** — What we gain, what we gave up, what has to stay consistent

ADRs are immutable once Accepted — if a decision changes, write a new ADR that
supersedes the old one and update the `Status` header on the original.
